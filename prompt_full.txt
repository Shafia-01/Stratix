## Fix 2: Correct interrupt() usage in planner_node and strategy_agent_node

File: src/graph/nodes.py

LangGraph's `interrupt(value)` is designed to:
1. Suspend the graph at that point
2. Return the human's response when the graph is resumed

Current code calls `interrupt({...})` but discards the return value, then reads `state["human_feedback"]` which must be injected externally. This works only if the API layer correctly injects feedback via `graph.update_state()` before resuming.

Verify the API layer (api/routes/agent.py) implements the resume flow as:
    graph.update_state(
        config,
        {"human_feedback": human_feedback_payload},
        as_node="planner_node"  # or "strategy_agent_node"
    )
    result = graph.invoke(None, config=config)

If the API does NOT do this, fix it to use `graph.update_state()` + `graph.invoke(None, config)` pattern.

Alternatively, capture the interrupt return value:
    human_input = interrupt({"checkpoint": "plan_approval", "research_plan": plan.model_dump()})
    # human_input now contains the human's feedback dict
    approved = human_input.get("approved", False)

Choose one pattern and apply it consistently to both interrupt sites.

## Task 4.5: Enhanced Confidence Scoring in aggregator_node

File: src/graph/nodes.py — aggregator_node()

Replace the current simple confidence heuristics with a structured rubric approach:

For `keyword_research`:
- 1.0 if count >= max_keywords AND avg(volume) > 0
- 0.7 if count >= max_keywords AND avg(volume) == 0 (Gemini fallback)
- scale linearly below max_keywords

For `serp_analysis`:
- 1.0 if organic_results >= 5 AND paa_questions >= 2
- 0.6 if organic_results >= 3
- 0.2 if organic_results < 3

For `competitor_gap`:
- 1.0 if opportunities >= 3 AND any gap_score > 70
- 0.5 if opportunities >= 1
- 0.0 if no opportunities

For `trend_forecast`:
- compute fraction of keywords with non-empty forecast_scores AND r_squared > 0.3

For `topic_cluster`:
- 1.0 if clusters >= 3 AND avg(keyword_count) >= 3
- 0.5 if clusters >= 2
- 0.2 if clusters == 1

Add `confidence_scores` to the `StrategyReport` dict before it enters state, so the LLM in `strategy_agent_node` receives them in context.
